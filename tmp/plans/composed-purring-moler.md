## Этап 3: Data migrations (бэкенд)

### 3.1 Замена "Да" → "Есть" (6 критериев)
**Новый файл:** `backend/catalog/migrations/XXXX_replace_da_with_est.py`

Критерии: Наличие ЭРВ, Регулировка оборотов вентилятора наруж. блока, Наличие подсветки экрана пульта, Держатель пульта ДУ, Функция дежурного обогрева +8С, Ароматизатор воздуха.

Для каждого:
1. В `custom_scale_json`: ключ "Да" → "Есть" (с тем же score)
2. В `ModelRawValue`: `raw_value="Да"` → `raw_value="Есть"`

### 3.2 Удаление вариантов значений (10 критериев)
**Новый файл:** `backend/catalog/migrations/XXXX_remove_scale_values.py`

| Критерий | Удалить из custom_scale_json |
|----------|------------------------------|
| Наличие обогрева поддона | "металлический тэн" |
| Мин. напряжение | "100", "150" |
| Ионизатор | "обычная щетка", "технологичный" |
| Приток свежего воздуха | "да" |
| Работа на обогрев | "нет", "да" |
| Максимальная длина фреонопровода | "-15", "-7 и выше", "-25" |
| Энергоэффективность | "2012", "2011", "2008" |
| Функция дежурного обогрева +8С | "А", "А++" |
| Гарантия | "Да", "Нет" |
| Ароматизатор воздуха | "3", "7", "5" |

**Решение пользователя:** если у модели уже стоит удаляемое значение — обнулить `raw_value` (очистить).
