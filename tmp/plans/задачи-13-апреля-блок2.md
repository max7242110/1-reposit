## Задача 1: Уникальные описания для каждой закладки

**Проблема:** Сейчас под таблицей одно общее описание из `methodology.description`. Нужно для каждого таба своё.

**Решение:** Добавить 3 текстовых поля в модель `MethodologyVersion` (описание для каждого таба), пробросить через API, показать на фронте.

### Шаги:

1. **Добавить поля в `MethodologyVersion`** — `backend/methodology/models.py`
   - `tab_description_index` — TextField, описание для таба "По индексу"
   - `tab_description_quiet` — TextField, описание для таба "Самые тихие"
   - `tab_description_custom` — TextField, описание для таба "Пользовательский рейтинг"

2. **Миграция** — `backend/methodology/migrations/`

3. **Обновить `MethodologySerializer`** — `backend/catalog/serializers.py`
   - Добавить 3 новых поля в fields

4. **Обновить TypeScript тип `Methodology`** — `frontend/src/lib/types.ts`
   - Добавить `tab_description_index`, `tab_description_quiet`, `tab_description_custom`

5. **Обновить `RatingPageContent.tsx`** — `frontend/src/components/RatingPageContent.tsx`
   - Заменить единый блок описания (строки 153-163) на условное отображение в зависимости от `activeTab`
   - standard → `methodology.tab_description_index`
   - quiet → `methodology.tab_description_quiet`
   - custom → `methodology.tab_description_custom`

### Файлы:

- `backend/methodology/migrations/` (новый)

## Задача 3: Пользовательский рейтинг — сортировка + кнопка

### 3a: Сортировка критериев по весу (от большего к меньшему)

**Текущее состояние:** `CustomRatingPanel.tsx` фильтрует `criteria.filter(c => c.is_public)` — порядок из API (по `display_order`).

**Решение:** Добавить `.sort((a, b) => b.weight - a.weight)` после filter.

### 3b: Переименовать кнопку и показывать условно

**Текущее состояние:** Кнопка "Сбросить к умолчанию" (строка 26), всегда видна.

**Решение:**
- Переименовать в "Включить все"
- Показывать только когда `enabledCount < publicCriteria.length` (хотя бы один выключен)

## Задача 4: SEO-friendly URL страницы модели

**Текущее:** `/v2/model/[id]` (по числовому ID)
**Нужно:** `/<brand>-<series>-<inner_unit>-<outer_unit>` (например `/Coolberg-City_on-off-CS-12SH1-IN-CS-12SH1-OUT`)

### Шаги:

1. **Добавить поле `slug` в `ACModel`** — `backend/catalog/models.py`
   - `slug = SlugField(max_length=500, unique=True, blank=True, db_index=True)`
   - В `save()` — автогенерация slug из brand.name, series, inner_unit, outer_unit
   - Функция транслитерации кириллицы, пробелы → `_`, части через `-`

2. **Утилита транслитерации** — `backend/catalog/utils.py` (новый)
   - Маппинг кириллица→латиница, замена пробелов на `_`

3. **Миграция** — schema + data (backfill slug для существующих моделей)

4. **Добавить slug в сериализаторы** — `backend/catalog/serializers.py`
   - `ACModelListSerializer` — добавить `slug` в fields
   - `ACModelDetailSerializer` — добавить `slug` в fields

5. **Добавить endpoint по slug** — `backend/catalog/urls.py` + `backend/catalog/views/ac_models.py`
   - Новый path: `models/by-slug/<slug:slug>/`
   - Или: изменить `ACModelDetailView` чтобы принимал и id, и slug

6. **Обновить TypeScript типы** — `frontend/src/lib/types.ts`
   - `ACModelSummary` и `ACModelDetail` — добавить `slug: string`

7. **Обновить API клиент** — `frontend/src/lib/api.ts`
   - Новая функция `getModelBySlug(slug: string)`

8. **Создать новый route** — `frontend/src/app/[slug]/page.tsx`
   - Страница модели по slug вместо `/v2/model/[id]`

9. **Обновить ссылки** — `frontend/src/components/RatingTableV2.tsx`
   - Заменить `href={/v2/model/${m.id}}` → `href={/${m.slug}}`

10. **Редирект со старого URL** — `frontend/src/app/v2/model/[id]/page.tsx`
    - redirect на новый URL по slug для обратной совместимости

### Файлы:
